# Git Workflow Guide for SmartCloset

## 🎯 Commit Strategy

### When to Commit
Commit after completing a **logical unit of work**, such as:
- ✅ Completing a feature from the checklist
- ✅ Fixing a bug
- ✅ Refactoring a component
- ✅ Adding tests
- ✅ Updating documentation

**Don't commit**: After every single file change or incomplete work

---

## 📝 Commit Message Format

Use **Conventional Commits** format:

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

### Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation only
- `style`: Code style changes (formatting, no logic change)
- `refactor`: Code refactoring
- `test`: Adding or updating tests
- `chore`: Maintenance tasks (dependencies, config)
- `perf`: Performance improvements

### Examples:
```bash
# Feature
git commit -m "feat(outfit): add mark outfit as worn functionality"

# Bug fix
git commit -m "fix(wardrobe): resolve image loading issue on iOS"

# Documentation
git commit -m "docs: update MASTER_CHECKLIST with outfit gaps"

# Multiple files
git commit -m "feat(outfit): create OutfitDetailsScreen with wear tracking

- Add OutfitDetailsScreen.tsx
- Update outfitService with wear tracking
- Add navigation to outfit details
- Update MASTER_CHECKLIST"
```

## 🔄 Standard Workflow

### 1. Check Status
```bash
git status
```

### 2. Review Changes
```bash
git diff
```

### 3. Stage Changes
```bash
# Stage all changes
git add .

# Or stage specific files
git add src/screens/OutfitDetailsScreen.tsx
git add MASTER_CHECKLIST.md
```

### 4. Commit
```bash
git commit -m "feat(outfit): add outfit details screen"
```

### 5. Push
```bash
git push origin main
# or your branch name
git push origin feature/outfit-details
```

---

## 🌿 Branching Strategy (Recommended)

### For New Features
```bash
# Create feature branch
git checkout -b feature/outfit-wear-tracking

# Work on feature...
# Commit changes...

# Push feature branch
git push origin feature/outfit-wear-tracking

# Merge to main when complete
git checkout main
git merge feature/outfit-wear-tracking
git push origin main
```

### For Bug Fixes
```bash
git checkout -b fix/image-loading-ios
# Fix bug, commit, push
```

### For Documentation
```bash
git checkout -b docs/update-checklist
# Update docs, commit, push
```

---

## ✅ Checklist Before Committing

- [ ] Code compiles without errors
- [ ] Tests pass (if applicable)
- [ ] No console.log() or debug code left behind
- [ ] MASTER_CHECKLIST.md updated if completing a task
- [ ] Code follows project style
- [ ] Commit message is descriptive

---

## 🚫 What NOT to Commit

- `node_modules/` (already in .gitignore)
- `.env` files with secrets
- Build artifacts (`ios/build/`, `android/build/`)
- IDE-specific files (`.vscode/`, `.idea/`)
- Personal notes or TODO files
- Incomplete/broken code

## 💡 Quick Commands

```bash
# See what changed
git status

# See detailed changes
git diff

# Stage everything
git add .

# Commit with message
git commit -m "type(scope): message"

# Push to remote
git push

# Undo last commit (keep changes)
git reset --soft HEAD~1

# See commit history
git log --oneline

# Create and switch to new branch
git checkout -b branch-name

# Switch branches
git checkout main
```

## 🔄 Daily Workflow

### Start of Day
```bash
git pull origin main
# Ensure you have latest changes
```

### During Development
```bash
# Commit after each logical unit of work
git add .
git commit -m "feat(scope): what you did"
```

### End of Day
```bash
# Push all commits
git push origin main

# Or if on feature branch
git push origin feature/your-branch
```

## 📊 Tracking Progress with Git

### View Recent Commits
```bash
git log --oneline -10
```

### See What Changed Today
```bash
git log --since="1 day ago" --oneline
```

### See Changes in a File
```bash
git log -p src/screens/OutfitScreen.tsx
```

## 🆘 Common Issues

### Forgot to Pull Before Pushing
```bash
git pull --rebase origin main
git push origin main
```

### Committed to Wrong Branch
```bash
git log  # Copy commit hash
git checkout correct-branch
git cherry-pick <commit-hash>
git checkout wrong-branch
git reset --hard HEAD~1
```

### Want to Undo Last Commit
```bash
# Keep changes
git reset --soft HEAD~1

# Discard changes
git reset --hard HEAD~1
```
